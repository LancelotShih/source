Wake Up Joe

An augmented reality voice powered morning routine experinece that assists in helping you get dressed, make breakfast, and update you on today's events.

Project Overview

A MakeUofT 2024 Hackathon project with the focus on home automation, we designed and developed "Wake Up Joe", an augmented morning routine experience that aimed to leverage the Qualcomm Android SDK to create a voice-powered assistant that helps users get ready for their day.

The routine would essentially begin when the user wakes up and wears their AR device of choice, which pulls up a UI overlay that guides them through various morning routine tasks such as selecting an outfit, making breakfast, weather concerns, and getting updates on the day's events.

Among these routines, an open mic in the room would listen for voice commands from the user to interact with the system, such as "Show me today's weather", "Brew me a cup of coffee", or "What's on my calendar today?". The system would then process these commands and provide relevant responses or actions through the AR interface.

The voice assistant was entirely local, built using Qualcomm's Android SDK using TensorFlow Lite models for speech recognition and natural language processing.

Challenges

One of the main challenges was making sure that the voice recognition worked well enough and could act upon requests well enough so that it was at least functional within the limited time frame of the hackathon. We had issues where it would misinterpret words, pick up gibberish and try to spell those words out, and also just missing commands entirely. By speccing a limited set of keywords and phrases that the system would recognize, we were able to improve accuracy significantly.

Another challenge was just getting the voice recognition to work on the Qualcomm SDK in the first place, as the device was fairly new and there was limited documentation available since it was meant to promote the new Snapdragon NPU unit for on-device AI processing. We had to do a lot of trial and error with different model formats and configurations before we were able to get a working setup.
