Farming Rover

A rover designed using various servo and hobby motors and remote controlled using HTTP requests. Designed to pick up objects via remote control and return to home base after usage.

Project Overview

Designed as a part of the UTRA Hacks 2024 Hackathon, the Farming Rover is a remote-controlled vehicle built to assist in agricultural tasks. The rover is equipped with various servo and hobby motors that allow it to navigate terrain, pick up objects, and return to its home base.

We used an ESP32 microcontroller as a means to handle all motor controls and an RPI as the main HTTP server that was used to receive commands from the user who would request for the rover to move in either preset routines or manual control via a web interface. There was also a live camera feed that allowed for the user to see in real time what the rover was seeing.

Challenges

One of the main challenges was figuring out how to tune the motors so it ran as intended, figuring out how to connect and control the robot wirelessly, and figuring out how to move the arm for crop gathering dexterity.

For instance, the grabber arms needed to close in the same fashion on both sides, otherwise the crop wouldn't be properly gripped and would slip out during the return journey. However, even if the motors closed the same way, there was the issue of the grabbers themselves not having enough grip to hold onto the crops. To solve this, we added rubber bands to the grabbers to increase friction and ensure a firm hold on the crops.

Lastly another big challenge was latency between what the server would see on the camera and what our user would see on their end. This is just inherently due to bandwidth restrictions of our hardware, but we tried to minimize this effect by offering those routines you can preset so that the user doesn't have to rely on constant camera feedback to move the rover.
