Project Overview

Designed as a part of the UTEK 2026 Hackathon, the VisiHelp is a wearable system providing comfortable, affordable, and accessible aid for the visually impaired. The device captures, transcribes, and reads aloud text from the environment using a combination of locally hosted web services and cloud-based OCR and TTS APIs.

We used an ESP32 CAM module as a means to handle all visual capture and would host a web server that would interact with a cloud server to process the image data. In the cloud, we used OpenAI's ChatGPT 4.1 Mini API to handle OCR and return a transcribed text version of the image. This text would then be sent to a Google Cloud TTS API backend that would convert the text to speech and send back an audio file that would be loaded onto another webserver. This second webserver is then accessed by the user through a mobile device, and included the functionalities of operating the camera, capturing images, and playing back the audio files.

The entire system was designed to be extremely low cost, with the only hardware really required being the ESP32 CAM module, and a custom headband inspired by smartwatch band designs to hold the camera module in place. The rest of the system is entirely software based, and can be accessed through any mobile device with internet access.

Challenges

The first main challenge we faced was getting the camera to actually take good photos. With the camera being manual focus, we needed to have the sensor capture enough data in the first place for the OCR to work in any capacity. This was eventually resolved through the use of the flash LED to brighten the environment and adjusting some of the various settings of the camera in software such as exposure, contrast, and max resolution to get the best possible image quality without sacrificing for slow loading times through file transfers.

The flash LED also posed a small issue as keeping it enabled for extended periods of time would cause the LED to overheat and start smoking. A timer had to be put in place to ensure the LED default state was always off, and would trigger outside of the times we needed to take a picture.

The cheap mounting solution was also a challenge as we wanted it to be worn similar to a pair of glasses. There was some experimentation in the beginning on where the camera mounting point should be, and whether glasses was an effective solution. In the end, we found that a headband provided a more stable and comfortable fit for longer periods of time. We ended up designing a headband out of a single piece of 8x10 in cardboard that would expand and shrink to multiple head sizes.

It should also be noted this entire project was built within the span of 5 hours. All planning, design, integration, and testing was done within this time, which meant across the 4 of us working on this project, trust and communication was key to ensure we were all on the same page and working towards the same goal.
